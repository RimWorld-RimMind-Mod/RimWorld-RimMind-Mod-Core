  RimMind — 让 RimWorld 的每个小人真正活过来
═══════════════════════════════════════════════

袭击结束。Alice 缩在角落发抖，她的好友刚刚倒在了虫族的利爪下。
你打开她的思维面板——她正在思考要不要离开这个该死的殖民地。
这不是脚本，不是随机事件。这是 AI 在那一刻为她做出的决定。

RimMind 做的事情很简单：把 LLM 接入 RimWorld 的每个小人。
但效果远不止"让小人说话"——它给每个 Pawn 一个完整的认知循环：

  感知(Perceive) → 思考(Think) → 行动(Act) → 记忆(Remember)

小人会注意到同伴的心情变化、殖民地的食物短缺、窗外的虫族逼近。
然后它们会真的思考这些事，做出自己的判断，甚至形成长期目标。

  核心体验
  ────────

  ▸ 每个 Pawn 都有独立思维
    PawnAgent 为每个小人维护独立的感知、目标栈和决策循环。
    没有两个小人会以相同的方式应对同一场危机。

  ▸ 上下文引擎让 AI 真正"懂"你的殖民地
    ContextEngine 用五层架构（静态→基线→环境→状态→历史）为 AI
    构建精准的上下文，不是把一堆数据丢给 LLM，而是只给它此刻
    最相关的信息。BudgetScheduler 确保每次请求都在 token 预算内。

  ▸ 飞轮自动调优，越用越聪明
    Flywheel 系统持续监控 AI 响应质量，自动调整温度、预算、权重
    等参数。你不需要手动调参——飞轮会替你找到最佳平衡点。

  ▸ 完整的感知桥接
    5 个 Harmony Patch 把 RimWorld 的原生事件（受伤、心情变化、
    社交互动、袭击、地图事件）翻译成 AI 能理解的感知信号。
    小人不是被动等待指令——它们会主动感知世界。

  子 Mod 生态
  ──────────

  RimMind-Core 是基础引擎，装上就能跑。但真正的魔法在子 Mod 里：

  ┌──────────────────┬──────────────────────────────────────┐
  │ RimMind-Actions  │ 让小人自主执行行动——工作、社交、逃跑  │
  │ RimMind-Advisor  │ AI 顾问，帮你分析殖民地状况并提建议   │
  │ RimMind-Dialogue │ 和小人/NPC 自由对话                   │
  │ RimMind-Memory   │ 长期记忆 + 暗记忆，小人记得一切       │
  │ RimMind-Personality│ 大五人格，每个小人有独特性格        │
  │ RimMind-Storyteller│ AI 叙事者，动态生成故事线           │
  └──────────────────┴──────────────────────────────────────┘

  推荐入门路径：Core → Dialogue → Personality → Memory
  想要更深的沉浸感再加 Actions 和 Storyteller。

  常见问题
  ────────

  Q: 需要什么 AI 服务？
  A: 任何 OpenAI 兼容的 API（OpenAI / 本地 Ollama / 其他兼容服务）。
     也支持 Player2 模式——你自己扮演 NPC 的 AI。

  Q: 会不会很贵？
  A: 默认配置下每个小人每 500 秒思考一次，token 用量可控。
     Flywheel 会自动优化参数，避免浪费。

  Q: 和 Vanilla Psychology / Psychology 等 mod 冲突吗？
  A: 不冲突。RimMind 通过 Harmony Patch 桥接，不修改游戏原生逻辑。

  Q: 可以只装 Core 不装子 Mod 吗？
  A: 可以。Core 自带完整的认知引擎和设置界面，但小人的行为需要
     子 Mod 来驱动。建议至少搭配 Dialogue。

  技术架构速览（给 Mod 开发者）

  - ContextEngine: L0(静态) → L1(基线+Diff) → L2(环境) → L3(状态) → L4(历史)
  - 注册 Provider: ContextKeyRegistry.Register(key, layer, priority, provider, modId)
  - 公共 API: RimMindAPI 静态类，所有子 Mod 通过此 API 与 Core 交互
  - 请求队列: AIRequestQueue (GameComponent)，支持优先级、冷却、取消
  - 详细文档: 见 AGENTS.md
